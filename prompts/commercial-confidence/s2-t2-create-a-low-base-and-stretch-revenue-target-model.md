# Create a low, base and stretch revenue target model

**Sprint:** Sprint 2: Commercial Confidence

**Lane:** Commercial Confidence

**Use this model or tool:** ChatGPT Project. DeepSeek for challenge. Claude for stakeholder-ready version.

**Use this asset:** Commercial Director Advisory Board: forecasting scorecard, strategy teardown. Executive Growth System: Revenue_Leak_Register.md

## Starter prompt

```text
Build a low, base and stretch revenue target model for this event or product.

Inputs I can provide: last-year actuals, current pipeline, price changes, sales capacity, churn risk, inventory, expected conversion, discounting and confidence rating.

Output for each scenario:
1. Revenue target.
2. Assumptions.
3. Risk.
4. Data gaps.
5. What would make it fail.
6. Decision leadership must make.

Constraint: do not invent missing numbers. Use placeholders and ask for data.
```

## Human starter action

Pick one event. Gather last-year revenue, current pipeline, average deal size, inventory and sales owners. Fill gaps with placeholders.

## Expected output

A target model that separates evidence from pressure.

## Success test

One target can be defended without relying on emotion or pressure.

## Friday review question

Which target assumption is weakest?

## Kill rule

If it cannot be explained in 5 bullets, simplify it.
