# Create the Stop Doing Register

**Sprint:** Sprint 1: Relief and Control

**Lane:** NEXT.io Control

**Use this model or tool:** ChatGPT Project for structure. Claude Project if you want to use Microsoft 365 examples.

**Use this asset:** Stuart Executive Growth System: Stop_Doing_Register.md, Work_Intake_Rules.md, Delegation_Responses.md

## Starter prompt

```text
Context: I am Commercial Director at NEXT.io. I keep getting pulled into CRM execution, HubSpot imports, list building, forms, sales collateral firefighting, and middleman communication. I need to protect leadership time without abandoning standards.

Existing assets: Stop Doing Register, Work Intake Rules, Delegation Responses.

Task: Create my Stop Doing Register.

Output format: table with columns: task, why it is a trap, correct owner, standard I still own, handoff message, escalation path, first date to enforce.

Constraints: no motivation, no giant operating system, no vague delegation language.

Success test: I can redirect at least one task in the next 48 hours.
```

## Human starter action

Open a blank table. List every task you did this week that someone else should own. Mark each as stop, delegate, supervise, automate or keep.

## Expected output

A written list of tasks you stop, delegate, standardise, automate or keep.

## Success test

10 tasks classified. One task redirected.

## Friday review question

Which executional task did I stop owning?

## Kill rule

If it becomes a complaint list rather than handoff language, kill the draft and redo it.
